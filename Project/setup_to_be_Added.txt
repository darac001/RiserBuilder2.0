For a designer, we want the workflow to be simple. Ideally they should not know anything about AutoLISP files.

A possible workflow:

One-time setup (IT / CAD manager)

Install Riser Builder:

C:\RiserBuilder\
│
├── main.lsp
├── config.lsp
├── input_parser.lsp
├── data_model.lsp
├── libraries.lsp
│
└── libraries\
    ├── device_library.csv
    └── panel_library.csv

Configure AutoCAD:

Option 1:

Add C:\RiserBuilder\ to AutoCAD Support File Search Path

Option 2 (better):

Create a shortcut/tool palette button that loads:
C:\RiserBuilder\main.lsp

Designer never touches this.

Daily workflow for designer
1. Create input spreadsheet

Designer opens the Excel template.

Example:

Project_Input.xlsx

Fills:

Door ID	Panel ID	Panel Type	Devices
101	ACP-01	ACP	CR,DC,ES
102	ACP-01	ACP	CR,ES
201	ACP-02	ACP_PS	CR,RX
2. Save CSV

Save as:

Project123_Riser_Input.csv

Inside project folder:

Project123/
│
├── Project123.dwg
├── Project123_Riser_Input.csv
└── Project123_Input.xlsx
3. Open AutoCAD drawing

Designer opens:

Project123.dwg
4. Run command

Example:

RB_BUILD

Tool opens:

Select Riser Input CSV:

Designer selects:

Project123_Riser_Input.csv
5. Tool runs automatically

Behind the scenes:

CSV
 |
 v
input_parser.lsp
 |
 v
raw-data
 |
 v
data_model.lsp
 |
 v
system-data
 |
 v
layout_engine.lsp
 |
 v
AutoCAD drawing
6. Result

AutoCAD creates:

ACP blocks
horizontal riser lines
door branches
device blocks
cable labels
Maintenance

If a new device is added:

Example:

BI  Biometric Reader

CAD manager only updates:

device_library.csv

No code changes.

So the designer's required knowledge is only:

Fill Excel template
Export CSV
Run RB_BUILD
Select CSV

Everything else is hidden.

That is the workflow I would aim for. It keeps the tool usable by someone who does not know AutoLISP.
